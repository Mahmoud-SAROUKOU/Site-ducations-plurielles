🔐 AUTHENTIFICATION GITHUB
─────────────────────────────────────────────────────────────

Lors du push, GitHub va vous demander de vous authentifier :

Option 1 : Navigateur (RECOMMANDÉ)
→ Une fenêtre de navigateur s'ouvrira automatiquement
→ Connectez-vous avec vos identifiants GitHub
→ Cliquez "Authorize"

Option 2 : Token Personnel (si demandé dans terminal)
→ Créez un token : https://github.com/settings/tokens
→ Cliquez "Generate new token (classic)"
→ Sélectionnez "repo" (toutes les cases)
→ Copiez le token généré
→ Utilisez-le comme mot de passe dans le terminal

🆘 EN CAS DE PROBLÈME
─────────────────────────────────────────────────────────────

❌ "remote origin already exists"
   → & "C:\Program Files\Git\bin\git.exe" remote remove origin
   → Puis réessayez la commande remote add

❌ "failed to push some refs"
   → Vérifiez que le dépôt GitHub est vide (pas de README)
   → Si vous avez coché README, supprimez le dépôt et recréez-le

❌ "Permission denied"
   → Créez un Personal Access Token sur GitHub
   → Utilisez-le comme mot de passe

❌ "Authentication failed"
   → Utilisez HTTPS (pas SSH) pour l'URL
   → Format: https://github.com/USERNAME/repo.git
